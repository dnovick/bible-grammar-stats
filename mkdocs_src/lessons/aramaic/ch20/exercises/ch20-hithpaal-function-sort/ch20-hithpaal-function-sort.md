# Chapter 20 — Hithpaal Semantic Function Sorting

*Basics of Biblical Aramaic, Van Pelt — Ch20: Reflexive, Reciprocal, Iterative, and Estimative Uses*

---

## Instructions

For each numbered Hithpaal/Ithpaal verbal form drawn from Daniel or Ezra, provide:

1. **Function code** — classify the Hithpaal's semantic function:
   - **RF** — Reflexive: the subject acts on itself
   - **RC** — Reciprocal: a plural subject acts on each other
   - **IT** — Iterative/Frequentative: the action is repeated or ongoing
   - **ES** — Estimative/Tolerative: the subject is reckoned as something, or permits an action
2. **Conjugation** — Perfect, Imperfect, or Participle
3. **PGN** — person (1/2/3), gender (m/f/c), number (s/pl)
4. **Root** — the three root consonants
5. **Translation** — an English rendering that reflects the function

**Function codes defined:**

| Code | Name | Definition |
|---|---|---|
| RF | Reflexive | Subject performs the action upon itself (e.g., "he washed himself") |
| RC | Reciprocal | Plural subject performs the action mutually (e.g., "they saw each other") |
| IT | Iterative/Frequentative | Action is repeated or continuously ongoing (e.g., "he kept walking") |
| ES | Estimative/Tolerative | Subject regards itself as something, or allows/endures an action (e.g., "it was reckoned as") |

**Morphological key:**
- Hithpaal Perfect: הִתְ‑ prefix before root (metathesis: הִשְׁתַּ‑ when R1 is sibilant)
- Hithpaal Imperfect: יִתְ‑ prefix (metathesis: יִשְׁתַּ‑ for sibilant R1)
- Hithpaal Participle: מִתְ‑ prefix

## Discussion Questions

1. Biblical Aramaic Hithpaal (הִתְ‑ prefix) corresponds to Hebrew Hithpael. Compare the Aramaic prefix pattern with Hebrew: does Aramaic preserve the sibilant metathesis when R1 = שׁ? Examine items 1, 2, 7, and 17 (roots with sibilant initials). What do you observe?

2. Items 18 (הִשְׁתַּכַּח = "was found") and 4 (הִתְחַשַּׁב = "was reckoned") function like passives in translation. How is the Hithpaal being used passively here, and how does this differ from the Peil (true passive)?

3. The IT (Iterative) items (3, 6, 10, 13, 16, 20) are all participles. Is this a coincidence, or does the iterative function especially suit the Aramaic participle tense-aspect? Explain by considering what the Aramaic participle expresses.

### Answers to Discussion Questions

**1.** Yes, Biblical Aramaic fully preserves sibilant metathesis, just as Hebrew does. When R1 is a sibilant (שׁ, ס, צ, שׂ), the Hithpaal tāw prefix and R1 switch positions:

- Items 1, 2, 17 (root שׁחה): הִתְ + שׁ → הִשְׁתַּ‑; יִתְ + שׁ → יִשְׁתַּ‑
- Item 7 (root שׁדל): אִתְ + שׁ → אִשְׁתַּ‑ (variant אִתְ‑ prefix)
- Item 18 (root שׁכח): הִתְ + שׁ → הִשְׁתַּ‑

In all cases, the result is that the sibilant precedes the tāw, producing a ‑שְׁתַּ‑ cluster. This is identical in mechanism to Hebrew Hithpael metathesis (e.g., הִתְשַׁמֵּר → הִשְׁתַּמֵּר). The process operates in both languages for the same phonological reason: the sequence tāw-sibilant is phonotactically marked, so the sibilant moves to before the tāw.

**2.** The Hithpaal in items 18 and 4 functions as a **passive** in translation, but through the reflexive/tolerative mechanism rather than a true structural passive. The Peil (e.g., כְּתִיב "it was written") is a dedicated passive stem — it lacks any reflexive morphology and simply encodes external agency acting on the subject. The Hithpaal in "was found" (הִשְׁתַּכַּח) is reflexive-passive: idiomatically, the subject "found itself" (i.e., was found), presenting the situation as if the subject yielded to being found. Item 4 (הִתְחַשַּׁב = "was reckoned") is estimative: the subject is reckoned or counted as something, with the Hithpaal expressing the subject's being categorized. Neither is a true structural passive; both derive their passive-like reading from reflexive or estimative semantics applied to verbs of perception/cognition that naturally produce a passive-sounding translation.

**3.** It is not a coincidence. The Aramaic **participle** (active or Hithpaal) expresses **durative, ongoing, or progressive** aspect — it describes an action as currently in progress or as a habitual state. The **iterative/frequentative** function (repeated action over time) overlaps almost perfectly with the participle's aspectual profile. When the Hithpaal participle is used iteratively, it expresses an action that recurs repeatedly or is continuously in process — which is exactly what the participle does aspectually. In contrast, the Hithpaal Perfect (past, completed) or Imperfect (future event) are less naturally iterative because they encode bounded actions. The participle's unbounded, progressive aspect makes it the natural vehicle for expressing that an action keeps happening or is a characteristic activity of the subject.
